# qa

이 폴더는 QA 체크리스트와 검증 메모를 저장한다.

고객사 전체 프로젝트(엔드투엔드) 흐름에서 **단계 5**는 PRD AC·acceptance test 기준 **전체 실행**과 미자동화 AC 보완을 따른다 (`.cursor/skills/client-project-lifecycle/SKILL.md` 단계 5). Gate 2 직후 ATDD-lite: [`atdd-lite.md`](atdd-lite.md), AC 템플릿: [`acceptance-criteria.template.md`](acceptance-criteria.template.md), 착수 체크: [`stage3-entry-checklist.md`](stage3-entry-checklist.md) §3d. 제품 첫 소비 증거: [`atdd-lite-consumption-checklist.md`](atdd-lite-consumption-checklist.md), 기록 예시: [`atdd-lite-consumption-record-example.md`](atdd-lite-consumption-record-example.md).  
선택 **리뷰어 GATE** 루브릭은 `reviewer-gate-rubric.md`를 참고한다.

예시:
- 기능별 QA 체크리스트
- 배포 전 점검표
- 회귀 테스트 메모
- 모바일/웹 확인 항목

권장 파일명 예시:
- feature-checklist.md
- release-checklist.md
- admin-table-qa.md
- stage3-entry-checklist.md
- atdd-lite-consumption-checklist.md
- atdd-lite-consumption-record-example.md
- verify-{날짜 또는 slug}.md (qa-agent 검증 산출)
