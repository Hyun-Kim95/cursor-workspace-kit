# ux-states — 앱 업데이트 UI

`65-design-gate`·`product-ui-core-global`에 맞춘 **최소 상태 스펙**. 이중 목업 범위에서는 **권장·강제** 2화면을 우선 준비한다.

## 공통

| 상태 | 설명 | UX |
|------|------|-----|
| 기본 | API 호출 전·`none` | 일반 앱 화면 |
| 로딩 | 버전 API 요청 중 | 전역 또는 스플래시 위 비차단 인디케이터 (정책 선택) |
| 오류 | API 실패 | PRD의 fail-open/closed — 조용히 통과 또는 짧은 안내 |

## recommended (권장)

| 항목 | 스펙 |
|------|------|
| 레이아웃 | 다이얼로그 / 바텀시트 (팀 선택) |
| 닫기 | **가능** — 「나중에」·X |
| 주 액션 | 「업데이트」→ `storeUrl` |
| 재표시 | PRD 정책 (세션·N일·버전당) |
| 접근성 | 제목·본문·버튼 라벨 명시, 아이콘만 사용 금지 |

## required (강제)

| 항목 | 스펙 |
|------|------|
| 레이아웃 | 풀스크린 또는 모달 (배경 탭·뒤로가기 **차단**) |
| 닫기 | **불가** |
| 주 액션 | 「업데이트」→ `storeUrl` |
| 부가 | 앱 주요 네비게이션 사용 불가 |

## 스토어 열기 실패

| 상태 | UX |
|------|-----|
| storeUrl 오류 / 스토어 미설치 | 오류 메시지 + URL 복사 또는 재시도 (required면 여전히 닫기 불가) |

## 반응형·다크모드

- 모바일 세로 기본; 태블릿은 동일 패턴 유지 가능
- 라이트/다크 모두 대비·버튼 가시성 확인 (`40-dark-mode`)
